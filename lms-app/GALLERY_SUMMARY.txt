================================================================================
  DATABASE SCHEMA
================================================================================

Table: gallery_settings

Columns:
  - id (primary key)
  - google_photos_album_url (string) - Shared album URL
  - album_id (string) - Auto-extracted album ID
  - is_enabled (boolean) - Enable/disable gallery
  - title (string) - Gallery title
  - description (text) - Gallery description
  - use_google_photos (boolean) - Toggle Google Photos vs local
  - created_at (datetime)
  - updated_at (datetime)

Note: Uses singleton pattern (only one record exists)

================================================================================
  GOOGLE PHOTOS URL FORMATS SUPPORTED
================================================================================

The system automatically extracts album IDs from these formats:

1. https://photos.app.goo.gl/ALBUM_ID
2. https://photos.google.com/share/ALBUM_ID
3. https://photos.google.com/u/0/album/ALBUM_ID

Example:
  Input:  https://photos.app.goo.gl/XYZ123ABC456
  Output: Album ID = XYZ123ABC456

================================================================================
  TROUBLESHOOTING
================================================================================

PROBLEM: Google Photos not showing
SOLUTION:
  1. Verify album is publicly shared
  2. Check URL is complete and correct
  3. Test URL in new browser tab
  4. Verify album ID was extracted (visible in admin)

PROBLEM: Local photos not appearing
SOLUTION:
  1. Check photos are in public/gallery/
  2. Verify file extensions (.jpg, .png, etc.)
  3. Ensure "Use Google Photos" is unchecked
  4. Check file permissions (must be readable)
  5. Refresh gallery page

PROBLEM: Gallery not accessible
SOLUTION:
  1. Verify "Enable Gallery" is checked
  2. Check Rails server is running
  3. For student/teacher views, verify logged in
  4. Check routes: bin/rails routes | grep gallery

PROBLEM: 404 Error
SOLUTION:
  1. Verify routes are configured
  2. Check controller files exist
  3. Restart Rails server
  4. Check logs: tail -f log/development.log
